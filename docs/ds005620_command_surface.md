# DS005620 Command Surface Reference

## 1. Command Classes

Commands are divided into four classes based on safety profile and whether they
require human authorization.

| Class | Description | Auto-run in CI? |
|---|---|---|
| A — Mock / CI safe | Run fully offline with in-tree fixtures; no real data | Yes |
| B — Planning / inspection | Read-only gate checks; produce plans, not executions | Yes (read-only) |
| C — Ontology / language governance | Validate claim language and ontology artifacts | Yes |
| D — Human / manual only | Real execution with real data; require explicit human authorization | **Never** |

## 5. Manual-Only Real Execution Command (Class D)

> **⚠ WARNING: This command must never be run automatically by any CI target, agent, or Makefile rule.**
>
> Running the benchmark with `--execute --peer-reviewed-contract-confirmed` on real data
> constitutes an empirical benchmark run. It requires a human operator to have:
> 1. Verified all prerequisites (see `make ds005620-real-operator-check`).
> 2. Reviewed the peer-reviewed label contract (`p12_external_contract.json`).
> 3. Confirmed the reviewed-label contract matches the operator's understanding.

Generated by the P18.3 gate for **manual** human execution only:

```bash
# MANUAL EXECUTION ONLY — do not automate
python -m sciencer_d.btc_icft.pipelines.run_ds005620_real_benchmark \
  --execute \
  --peer-reviewed-contract-confirmed \
  --contract outputs/btc_icft/ds005620_reviewed_contract/p12_external_contract.json \
  --metadata data/DS005620/events.tsv \
  --signal-blocks outputs/btc_icft/eeg_signal_blocks_from_mne/DS005620 \
  --level-m outputs/btc_icft/eeg_level_m/DS005620/features_m_signal.csv \
  --level-t outputs/btc_icft/eeg_level_t/DS005620/features_t_signal.csv \
  --out outputs/btc_icft/ds005620_real_benchmark_execution
```

No Makefile target should auto-run the real benchmark with `--execute --peer-reviewed-contract-confirmed`.

## 8. Commands That Must Not Be Automated

The following behaviors must never be implemented in CI, agents, or automated pipelines:

| Forbidden automation | Reason |
|---|---|
| Running `--execute --peer-reviewed-contract-confirmed` on real DS005620 data | Requires human review of peer-reviewed label contract |
| Downloading DS005620 data automatically | Data provenance must be verified by a human |
| Promoting ontology candidates (C/Q/O) automatically | Requires independent replication + human out-of-band review |
| Modifying reviewed label contracts or activation declarations | These are human-authored artifacts |
| Generating baseline waivers for unsafe language in publication artifacts | Generated outputs must be fixed, not waived |
| Bypassing the real/local execution gate | Gate exists to enforce human review step |

## 9. Recommended Local Operator Sequence

For a human operator performing a first real/local DS005620 benchmark run:

```
Step 1:  Obtain DS005620 events.tsv → place at data/DS005620/events.tsv
Step 2:  make ds005620-real-operator-check   (verify prerequisites; expect blockers)
Step 3:  python -m sciencer_d.btc_icft.pipelines.validate_activation_declaration ...
         (P17.0: validate human-authored declaration)
Step 4:  python -m sciencer_d.btc_icft.pipelines.materialize_ds005620_reviewed_contract ...
         (P17.1: materialize reviewed contract)
Step 5:  python -m sciencer_d.btc_icft.pipelines.preflight_eeg_readers --dataset-id DS005620 --root data/DS005620 ...
         (P19.0: preflight EEG reader capabilities)
Step 6:  python -m sciencer_d.btc_icft.pipelines.extract_mne_signal_blocks --dataset-id DS005620 --input ... 
         (P19.1: MNE extraction; requires mne installed)
Step 7:  python -m sciencer_d.btc_icft.pipelines.convert_mne_to_signal_blocks --dataset-id DS005620 --mne-extract ...
         (P19.2: convert to canonical signal blocks)
Step 8:  python -m sciencer_d.btc_icft.pipelines.run_ds005620_level_m_signal_features ...
         (P9: Level M signal features)
Step 9:  python -m sciencer_d.btc_icft.pipelines.run_ds005620_level_t_signal_topology ...
         (P10: Level T topology telemetry)
Step 10: make ds005620-real-operator-check   (re-check; all blockers should be resolved)
Step 11: HUMAN REVIEW: read generated execution gate plan; verify reviewed contract labels
Step 12: MANUAL: run generated real execution command (do not automate)
Step 13: make ds005620-autonomy-check        (validate outputs; ontology check)
Step 14: make ds005620-generated-artifact-check (language gate on all generated outputs)
```
